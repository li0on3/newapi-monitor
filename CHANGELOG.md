# Changelog

[简体中文](CHANGELOG.md) | [English](CHANGELOG_EN.md)

## Unreleased

## 1.2.2 - 2026-07-23

### Fixed

- 兼容 OpenAI 官方状态接口在没有活动事件时省略 `incidents` 字段的响应，避免官方状态采集器持续失败并将平台健康检查误判为异常。

## 1.2.1 - 2026-07-22

### Changed

- 将 OpenAI 官方状态从总览大型区域迁移到独立的 `/monitor/upstream-status` 页面，总览仅保留可点击的轻量参考提示，渠道卡片重新成为首要信息。
- 将“本地真实渠道”明确设为主要判断信号，将 OpenAI 全局状态设为辅助上下文；无关官方事件不再表现为本地渠道故障。
- 即使管理员选择让官方状态影响总览，也只有已关注的业务组件异常才会影响 `OVERALL STATUS`。
- 优化桌面、窄屏和移动端导航布局，新增页面不会导致菜单换行重叠。

## 1.2.0 - 2026-07-22

### Added

- 接入 OpenAI 官方状态页，定时采集平台状态、组件状态、活动事件及官方进展时间线。
- 总览新增独立的“OpenAI 官方状态”区域；默认不参与本地渠道总览状态，避免把上游公告误判为本地故障。
- 系统配置新增“上游官方状态”页面，可配置采集频率、超时、告警影响级别、连续确认次数、组件范围和角色可见性，并支持实时连接测试。
- 官方事件与本地 OpenAI 模型渠道健康度关联展示，事件中心支持官方事件筛选、详情和恢复依据。

### Security

- 官方状态采集仅访问代码内固定的 HTTPS 官方接口，并限制响应体大小、超时和数据结构，避免可配置 URL 引入 SSRF 风险。

## 1.1.0 - 2026-07-22

### Added

- 可校验 SHA-256 的 Linux 一键安装器，默认使用固定版本 GHCR 多架构镜像并仅监听回环地址。
- 首次启动初始化向导：一次性令牌 15 分钟有效，支持 New API 管理员凭据自动换取令牌或手工填写已有令牌。
- `monitorctl` 运维命令，覆盖状态、日志、诊断、在线备份、升级、回滚、重置紧急管理员、续期初始化令牌和安全卸载。
- GitHub Release 自动附带安装器、部署包和校验文件。

### Security

- New API 管理员密码只在初始化请求内存中用于换取令牌，不写入数据库、日志或响应。
- 初始化完成后接口自动关闭，并使用令牌哈希、有效期和失败限速降低未配置实例暴露风险。

### Added

- 渠道同步、渠道探测、日志和资源采集的新鲜度自检。
- 采集器异常与恢复事件、详细运行状态页面和 503 健康降级。
- 数据库敏感配置加密、Host 白名单和状态变更请求校验。
- 非 Root 容器、资源限制、只读文件系统和 Docker Socket Proxy 加固。
- `manage.py` 初始化、部署诊断和 SQLite 在线备份。
- GitHub Actions、CodeQL、Dependabot、贡献和安全文档。
- 事件调查中心：状态、级别、类型、时间和全文筛选，事件队列与调查详情双栏工作区，以及恢复时长统计。
- 事件同时保留原始触发原因和恢复依据，支持历史数据兼容迁移。
- 系统配置新增“总览展示”，可分别控制管理端与普通用户看到的渠道，并按角色独立计算总览状态。
- 企业微信自建应用、企业微信群机器人、飞书自建应用、飞书群机器人和邮件统一通知分发。
- 通知中心支持动态配置、秘密字段加密和每个渠道独立触发真实测试告警。
- 按 API Key 查询真实用量、模型限制和最近调用详情，并提供角色与频率控制。
- 前端使用独立 History API 路由，支持深层链接、刷新和浏览器前进后退。
- 中英文 README、贡献指南、安全策略和路线图。
- Git tag 自动生成 GitHub Release，并发布带 SBOM/provenance 的 GHCR 多架构镜像。

### Fixed

- 渠道探测改为独立工作线程并支持有限并发，避免慢探测阻塞日志、资源和渠道同步采集。
- 渠道故障与恢复增加连续次数确认；临时 5xx、429 和超时先按降级告警，减少单次波动造成的告警抖动。
- 多渠道同时出现相同认证或分组权限错误时合并为探测凭证事件，不再误报为多个上游渠道故障。
- 渠道同步工作线程直接记录采集新鲜度，避免主循环未及时消费队列造成错误的采集器过期告警。
- New API 使用日志分页参数改为 `p`，避免回填和高流量场景漏日志。
- 渠道卡片时间与跳转箭头重叠。
- 总览健康状态、请求统计和渠道事件严格按界面启用的渠道范围计算。
- 运行中的容器会自动清理历史遗留的错误事件，避免状态误报。
- 普通 New API 用户默认映射为只读总览角色，日志、资源、事件和配置接口继续执行服务端权限校验。
- 修复事件恢复时覆盖原始告警内容，导致无法解释“为什么发生”的问题。
